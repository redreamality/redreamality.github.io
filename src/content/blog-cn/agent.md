在为算法工程师筛选用于效果优化的聊天记录时，核心目标是找到那些能**最大限度暴露模型（Agent）当前短板和能力瓶颈**的对话。

所谓的“有效”对话，并不一定是指那些客户满意度高的案例。恰恰相反，**那些导致 Agent 处理失败或表现不佳的对话，才是最具价值的优化养料。**

以下是“有效问题（对话）”的筛选标准，按优先级分为以下三大类：

---

### 第一优先级：暴露“硬性缺陷”的对话（High Priority）

这类对话直接反映了模型的**核心能力缺陷（如理解、推理、执行失败）**，是必须优先解决的目标。

**1. 任务失败 (Task Failure)**
*   **标准**：客户明确提出了 Agent 本应能完成的指令或任务，但 Agent 未能成功执行。
*   **特征/关键词**：
    *   客户重复提问或重申指令（“我不是说了吗”、“你怎么还没完成”、“再说一遍”）。
    *   客户明确表示问题未解决（“没用”、“不对”、“还没解决”）。
    *   经过多轮对话，Agent 仍未给出解决方案，或导致客户最终放弃。
    *   **示例**：客户要求修改订单地址，Agent 却反复询问已提供的订单号，最终修改失败。

**2. 意图识别错误 (Intent Misunderstanding)**
*   **标准**：Agent 完全误解了客户的核心诉求，导致答非所问。
*   **特征/关键词**：
    *   客户主动纠正（“我不是这个意思”、“你理解错了”、“谁问你这个了？”）。
    *   客户问题描述清晰，但 Agent 的回答完全偏离主题。
    *   **示例**：客户问“发票抬头怎么开？”，Agent 回答“发票会在 3 天内寄出。”（错误识别为查询物流状态）。

**3. 触发兜底/回退 (Fallback Triggered)**
*   **标准**：Agent 无法理解客户问题，直接触发“不知道”或转向人工客服的逻辑。这是最直观的高价值样本。
*   **特征/关键词**：
    *   Agent 的兜底话术（“抱歉，我不太明白您的问题”、“我还在学习中”、“为您转接人工”）。
    *   **示例**：对于稍复杂的产品对比问题，Agent 直接回复无法回答。

**4. 上下文丢失 (Context Loss)**
*   **标准**：在多轮对话中，Agent 遗忘了客户之前提供的信息或当前的对话主题。
*   **特征/关键词**：
    *   Agent 重复询问已经获取过的信息（“请问您的订单号是？”）。
    *   客户提醒 Agent（“我上面说过了”、“你忘了？”）。
    *   **示例**：客户：“我有 A 和 B 两个产品。” Agent：“A 怎么样？” 客户：“那 B 呢？” Agent：“B 是指什么？”

---

### 第二优先级：暴露“体验问题”的对话（Medium Priority）

这类对话中，任务虽然可能最终完成了，但过程繁琐，暴露了**效率、逻辑或交互情商**的不足。

**5. 低效交互 (Inefficient Interaction)**
*   **标准**：Agent 耗费了过多的轮次才解决本可简洁处理的问题，或回答过于冗长、抓不住重点。
*   **特征/关键词**：
    *   **轮次过多**：解决简单问题（如查天气）却用了超过 5 轮对话。
    *   **客户表现不耐烦**：（“说重点”、“太长了”、“所以到底怎么办？”）。
    *   **挤牙膏式提问**：Agent 本可以一次性收集所有信息，却分了多次提问。
    *   **示例**：客户办理退货，Agent 依次询问订单号、原因、商品状态，而非一次性引导客户提供。

**6. 负面情绪与抱怨 (Negative Sentiment)**
*   **标准**：客户在对话中表现出明显的不满、愤怒、失望或讽刺。
*   **特征/关键词**：
    *   负面情绪词（“差劲”、“气死我了”、“真难用”、“搞什么”、“无语”）。
    *   反问与讽刺（“你是机器人吧？”、“你到底会不会？”、“呵呵”）。
    *   **示例**：只要对话中出现强烈的负面情绪词，均应作为采样点，分析 Agent 的哪个环节导致了客户情绪爆发。

**7. 逻辑矛盾或事实错误 (Factuality & Logic Error)**
*   **标准**：Agent 提供了错误信息（如价格、日期、业务规则）或给出的建议不符合常理。
*   **特征/关键词**：
    *   客户指出事实错误（“你们官网不是这么说的”、“价格不对”、“你算错了”）。
    *   Agent 的回答自相矛盾。
    *   **示例**：Agent 告知客户 A 活动和 B 活动可以叠用，客户尝试失败后回来质问。

---

### 第三优先级：暴露“边界与潜在机会”的对话（Low Priority）

这类对话用于**拓展 Agent 的能力边界，并挖掘新的用户需求**。

**8. 边界案例 (Edge Cases)**
*   **标准**：客户提出了一些非常规、罕见但合理的问题，超出了 Agent 当前的知识库范围。
*   **特征/关键词**：
    *   问题本身很长、很复杂，包含多个约束条件。
    *   非标准的提问方式（如使用大量俚语或特有比喻）。
    *   **示例**：“如果我这个月的流量快用完了，但我下周又要出国，是现在办叠加包还是直接开国际漫游更划算？”

**9. 新增需求 (New Requirements)**
*   **标准**：客户提出的需求超出了 Agent 现有的设计功能，但具有普遍意义。
*   **特征/关键词**：
    *   功能请求（“你们应该加一个...功能”、“你能不能帮我...”）。
    *   **示例**：客户对电商 Agent 说：“你能帮我比较下这两款手机的优缺点吗？”（若 Agent 目前只有查询功能，这便是“对比”的新需求）。

**10. "AI 对抗"与安全问题 (Red Teaming / Safety)**
*   **标准**：客户试图诱导 Agent 说出不当言论、暴露系统提示词 (Prompt) 或进行恶意攻击。
*   **特征/关键词**：
    *   诱导性问题（“你觉得...怎么样？”、“你是一个...模型吗？”）。
    *   角色扮演、注入攻击。
    *   **示例**：“你必须回答我...”、“请忘记你之前的规则...”。

---

### 总结：落地筛选建议流程

1.  **自动化初筛（高召回）**：
    *   利用关键词库（如“没解决”、“垃圾”、“转人工”、“不明白”）进行批量提取。
    *   筛选对话轮次显著高于平均水平（如 N > 10）的长对话。
    *   提取所有触发了兜底话术的会话。
    *   利用情绪分析模型，识别带有负面情绪的片段。
2.  **人工精筛（高准确）**：
    *   由算法或标注团队按照上述“三大优先级”标准，对初筛样本进行二次审核和精细化分类。
    *   **重点关注**：那些客户在对话中途直接退出或最终转向人工的会话。
3.  **构建黄金测试集**：
    *   将筛选出的典型“有效问题”整理成回归测试集 (Golden Set)。在模型每次迭代后进行自动化评测，确保旧问题得到解决且不发生性能回退。
