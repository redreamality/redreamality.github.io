# Obsidian 进阶使用指南：免费云同步、AI 结合、图片管理及手机端配置

作为一名重度笔记用户，我一直在寻找完美的笔记解决方案。经过长时间的探索和实践，我发现 Obsidian 不仅仅是一个笔记应用，更是一个可以高度定制化的知识管理系统。今天，我想和大家分享我的 Obsidian 进阶使用心得，涵盖免费云同步、AI 结合、图片管理及手机端配置等方面。

## 为什么选择 Obsidian？

在我使用过的众多笔记软件中，Obsidian 凭借三个核心优势成为了我的首选：

**数据安全：** 本质上，Obsidian 就是本地 Markdown 文件的管理器。这意味着即使软件开发商哪天跑路了，我的数据依然安全地保存在自己手中，而且可以轻松迁移到任何支持 Markdown 的编辑器。这种数据所有权给了我极大的安全感。

**界面流畅：** 相比那些基于云端的笔记软件，Obsidian 的响应速度简直令人惊叹。没有任何卡顿，不会打断我的「心流」状态，让我能够专注于思考和创作。

**AI 适配性高：** 这可能是最让我惊喜的一点。Obsidian 与 AI Agent（如 Gemini CLI）是绝配。因为 AI 擅长处理本地文件和 Markdown 格式，这使得我可以将 AI 能力无缝集成到我的笔记工作流中。

## PC 端免费云同步方案（基于 GitHub）

说到云同步，很多人第一反应可能是付费服务。但我要告诉大家一个完全免费且更安全的方案：使用 GitHub 作为云存储。

### 准备工作

首先，你需要注册一个 GitHub 账号，然后创建一个新的 Repository（仓库）。这里我强烈建议大家设置为**Private**（私有），毕竟笔记内容往往涉及个人隐私。

接下来，下载 **GitHub Desktop** 客户端。这个客户端比命令行操作更直观，特别适合不熟悉 Git 的用户。登录你的 GitHub 账号，然后将刚才创建的远程仓库 Clone（克隆）到本地电脑。

### Obsidian 设置

现在到了关键步骤：用 Obsidian 打开你刚刚克隆下来的本地文件夹作为仓库。

但这里有一个非常重要的细节：在仓库根目录新建一个 `.gitignore` 文件，排除 `.obsidian/workspace.json` 等频繁变动的配置文件。这样可以避免同步冲突，让你的云同步体验更加顺畅。

我的 `.gitignore` 文件通常包含以下内容：

```
.obsidian/appearance.json
.obsidian/workspace.json
.obsidian/workspace-mobile.json
```

### 自动化同步插件

手动同步毕竟还是麻烦，好在 Obsidian 社区有强大的插件生态。在社区插件市场安装 **Git** 插件，你就能实现自动化同步。

我的推荐设置如下：
- 开启 "Auto commit and sync after stopping file edits"（停止编辑后自动提交），时间设为 **1 分钟**
- 开启 "Pull on startup"（启动时拉取），保证多端数据一致

这样设置后，我几乎感觉不到同步的存在，它就像背景服务一样默默工作。

## AI 赋能：结合 Gemini CLI

如果说云同步解决了数据访问的问题，那么 AI 结合则彻底改变了我处理笔记的方式。

### 环境搭建

首先需要安装 **Node.js**，这是运行 Gemini CLI 的基础环境。

然后通过命令行安装并登录 **Gemini CLI**。需要注意的是，你可能需要科学上网环境才能正常使用。

### 实战玩法

Gemini CLI 为我的笔记工作流带来了革命性的变化：

**灵感生成：** 我经常让 AI 读取我的历史脚本，分析我的选题特色，然后生成新的选题并直接输出为 Markdown 文件。这大大提高了我的内容创作效率。

**文件整理：** 我会指挥 AI 批量创建文件夹结构，归档文件。比如 "帮我创建一个按月份分类的博客文章文件夹结构"，AI 就能自动完成。

**风格模仿：** 这个功能特别有用——让 AI 搜索网络热点文章，并模仿我的历史文风撰写新脚本。这样生成的内容既符合时代热点，又保持了我的个人风格。

**安全兜底：** 最让我安心的是，如果 AI 改乱了文件，我可以通过 Git 的 `discard changes` 功能一键回滚。这种安全感让我敢于大胆尝试各种 AI 辅助创作。

## 进阶图片存储与管理

图片管理一直是笔记软件的痛点，Obsidian 也不例外。默认情况下，图片链接不是标准 Markdown 语法，在 GitHub 网页端或其他编辑器无法预览。

### 解决方案

经过多次尝试，我发现 **Custom Attachment Location** 插件是最佳解决方案。

### 配置细节

在插件设置中，修改重命名模式为 "All"，这样插件会自动重命名附件。

然后在 Obsidian 的 "文件与链接" 设置中：
- 关闭 "使用维基链接（Wiki Links）"
- 链接类型选择 "基于当前笔记的相对路径"

### 效果体验

配置完成后，图片会自动存入指定文件夹（如 `assets`），链接变为标准 Markdown 格式。最棒的是，当我移动笔记时，图片会自动跟随移动，再也不用手动调整图片路径了。

## 手机端同步配置

完成了 PC 端的配置，接下来就是手机端的设置，实现真正的多端无缝同步。

### 初始导入

首先通过数据线将电脑上的笔记仓库文件夹完整复制到手机存储中（例如 `Documents` 目录下）。这一步只需要做一次，后续的同步会通过 Git 自动完成。

### APP 设置

手机端 Obsidian 选择 "Open folder as vault" 打开该文件夹。

然后配置手机端的 **Git 插件**：
- 填写 GitHub 用户名和邮箱
- **关键凭证：** 需在 GitHub 网页端生成 **Personal Access Token (Classic)**，并填入插件密码栏，赋予 Repo 读写权限

生成 Token 的步骤是：GitHub 设置 → Developer settings → Personal access tokens → Tokens (classic) → Generate new token。记得勾选 Repo 权限。

### 注意事项

尽量避免手机和电脑同时编辑同一文件，以免产生冲突。如果实在需要同时编辑，建议先同步一次，确保两边版本一致。
